# Sliding Puzzle — Design Document

## Overview
- Sliding Puzzle is a single-player tile-sliding game on an N×N grid with one empty space. Tiles (numbers or pieces of a picture) start scrambled; the player slides tiles into the empty space to restore the correct order/image.
- It generalizes the classic 15-puzzle (4×4) to other sizes.
- Only solvable scrambles are dealt. There is no failure state; the puzzle is complete when ordered.
- Fully offline, single device, local stats only.
- Shared conventions: see [`common/puzzle-grid-board.md`](../../common/puzzle-grid-board.md), [`common/puzzle-controls.md`](../../common/puzzle-controls.md), [`common/puzzle-flow.md`](../../common/puzzle-flow.md).

## Settings
- **Size**: 3×3 (Easy), 4×4 (default, the "15-puzzle"), 5×5, 6×6.
- **Tile content**: Numbers (default) / Picture (uses a bundled image cut into tiles).
- **Show target**: on (default for picture) — a faint reference of the goal.
- **Blank position**: bottom-right (default) goal.

## Visual Style
- Material 3 surfaces, underwater palette from `ui/theme/Color.kt`.
- Tiles are rounded `Aqua2` blocks with large legible numbers, or image fragments; the empty cell shows `Dark0`.
- A correctly placed tile gets a subtle `Aqua1` accent so progress is visible.
- Sliding animates the tile gliding into the gap (≤150 ms, board-area only).

## How to Play
- Tap a tile adjacent to the empty space to slide it in.
- Keep sliding until the tiles are in order (or the picture is whole), with the blank in the bottom-right.

## Controls
- **Tap** a tile in the empty space's row/column: slides it (and tiles between it and the gap) toward the gap. See [`common/puzzle-controls.md`](../../common/puzzle-controls.md).
- **Swipe** toward the gap: equivalent multi-tile slide.
- **Undo**: revert the last slide.
- **Shuffle**: deal a new solvable scramble.

## Gameplay Rules
- Only tiles orthogonally in line with the empty space can move; a tap may slide a whole run toward the gap.
- Scrambles are generated by random legal moves (or parity-checked) to guarantee solvability.
- Solved = every tile in its goal position with the blank in its home corner.
- Solvability/parity, move application, and solved-check are pure functions in `controller/`, unit-tested.

## State Machine
A dedicated `SlidingPuzzleStateMachine` in `state/` exposes `StateFlow<SlidingPuzzleState>`.
```
Idle
 └─ StartGame → Playing (solvable scramble dealt)
Playing
 ├─ TileSlid → Playing (moves++)
 ├─ UndoRequested → Playing
 ├─ ShuffleRequested → Playing (new scramble)
 └─ BoardOrdered → Solved
Solved
 └─ NewGame → Playing
```
